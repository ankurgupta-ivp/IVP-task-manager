1
Set up your database (Supabase — free)
1
Create a free Supabase account
Go to supabase.com and click the green "Start your project" button. Sign up using your existing GitHub account — it's one click, no new password needed.
↗ Go to Supabase.com
2
Create a new project
After signing in, click the "New project" button. Fill in:
  1. Name: type anything, e.g. ivp-tasks
  2. Database Password: type a strong password (save it somewhere safe — you won't need it often)
  3. Region: choose ap-south-1 (Singapore) — closest to India
Click "Create new project" and wait about 60 seconds for it to set up.
✓ You'll see a dashboard appear. That means your database is ready.
3
Run the database setup script
This is the step that creates all the tables your app needs. It only takes 30 seconds.
  1. In your Supabase project, look at the left sidebar and click "SQL Editor" (it looks like a little code icon)
  2. Click the "New query" button (top left of the SQL editor area)
  3. Open the file called supabase-schema.sql from the zip you downloaded — open it in Notepad (right-click → Open with → Notepad)
  4. Press Ctrl+A to select all the text, then Ctrl+C to copy it
  5. Click inside the Supabase SQL Editor box and press Ctrl+V to paste
  6. Click the green "Run" button (or press Ctrl+Enter)
✓ You should see "Success. No rows returned" at the bottom. This means everything worked. Your database is fully set up with all your existing data intact.
⚠ If you see a red error message, it may mean you've already run this script before (from the HTML version). That's fine — your data is safe, just move on to Step 4.
4
Copy your two secret keys
You need two pieces of information from Supabase. Here's exactly where to find them:
  1. In Supabase, click the gear icon ⚙ at the very bottom of the left sidebar (it says "Project Settings")
  2. In the settings menu that opens, click "API"
  3. You'll see two things to copy:
Project URL → https://abcdefghijkl.supabase.co ← Copy this entire line anon / public key → eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzdXBhYmFzZSJ9... ← Copy this long key (starts with "eyJ")
ℹ Paste both into a Notepad file for now — you'll need them in Phase 2. Label them clearly so you remember which is which.
2
Edit 2 files (in Notepad — takes 5 minutes)
5
Unzip the downloaded file
Find the file ivp-tasks-react.zip that you downloaded. Right-click it and choose "Extract All…" (Windows) or double-click it (Mac). A folder called ivp-tasks will appear. Open that folder — keep it open, you'll need it for the next steps.
6
Edit File 1 — paste your Supabase keys
Inside the ivp-tasks folder, navigate to: src → lib → find the file called supabase.js
  1. Right-click supabase.js → Open with → Notepad
  2. You'll see these two lines near the top:
// BEFORE — what you see when you open the file: export const SUPABASE_URL = 'https://YOUR_PROJECT_ID.supabase.co' export const SUPABASE_ANON = 'YOUR_ANON_KEY' // AFTER — replace with your actual values from Step 4: export const SUPABASE_URL = 'https://abcdefghijkl.supabase.co' export const SUPABASE_ANON = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...'
  1. Delete YOUR_PROJECT_ID and type in your Project URL (keep the quotes and everything else)
  2. Delete YOUR_ANON_KEY and paste your anon key (keep the quotes)
  3. Press Ctrl+S to save. Close Notepad.
⚠ Keep the single quote marks ' around your values. Only replace what's inside the quotes, not the quotes themselves.
7
Edit File 2 — set your repository name
Back in the main ivp-tasks folder, find the file called vite.config.js. Right-click it → Open with → Notepad.
Find this line near the top:
// BEFORE: const GITHUB_REPO_NAME = 'ivp-task-manager' ← change this // AFTER (use whatever name you gave your GitHub repository): const GITHUB_REPO_NAME = 'my-actual-repo-name'
ℹ What is the repository name? It's the name you'll give your project on GitHub in Step 8. You decide the name — it can be anything like ivp-tasks or task-manager. Whatever name you choose in Step 8, type that exact same name here.
✓ Once edited, press Ctrl+S to save and close Notepad. Both files are now configured.
3
Publish to the web (GitHub — free)
8
Create a new GitHub repository
A "repository" is just a folder on GitHub that stores your project files.
  1. Go to github.com and sign in
  2. Click the + icon at the top-right of the page
  3. Click "New repository"
  4. In the "Repository name" box, type your chosen name — e.g. ivp-task-manager (use the same name you put in vite.config.js in Step 7)
  5. Make sure "Public" is selected
  6. Leave everything else as-is and click the green "Create repository" button
↗ Create repository on GitHub
9
Upload all your files to GitHub
After creating the repository you'll land on an empty page. Here's how to upload your files:
  1. On the empty repository page, click the link that says "uploading an existing file"
  2. A file upload area appears. Go back to your ivp-tasks folder on your computer
  3. Press Ctrl+A to select ALL files and folders inside ivp-tasks
  4. Drag and drop them all onto the GitHub upload area
  5. Wait for the upload to complete (you'll see file names appearing)
  6. Scroll down and click the green "Commit changes" button
⚠ Important: Upload the contents of the ivp-tasks folder — not the folder itself. When you drag, you should be dragging files like package.json, index.html, the src folder, etc. — not a single folder called ivp-tasks.
10
Turn on GitHub Pages
This tells GitHub to publish your app as a live website.
  1. In your GitHub repository, click the "Settings" tab (at the top of the page)
  2. In the left sidebar, scroll down and click "Pages"
  3. Under "Source", click the dropdown and select "GitHub Actions"
  4. Click "Save"
✓ That's it — GitHub will now automatically build and publish your app whenever files change. You don't need to do anything else for deployment.
11
Wait 2 minutes, then check your live app
GitHub is now building your app automatically. This takes about 2 minutes the first time.
  1. Click the "Actions" tab in your repository (at the top)
  2. You'll see a workflow running — it has a yellow spinning circle ●
  3. Wait until it turns into a green checkmark ✓
  4. Go back to Settings → Pages
  5. You'll see a green box saying "Your site is live at…" with your URL
Your live app URL will look like: https://your-github-username.github.io/ivp-task-manager Example: https://ankurgupta.github.io/ivp-task-manager
ℹ Bookmark this URL and share it with your team. Anyone who visits it will see the login screen and cannot get in without credentials you create.
12
Log in and add your team members
Open your live URL in a browser. Log in with your admin account:
Email: ankurgupta@ivp.in Password: changeme123
⚠ Change your password immediately after logging in. Click your name in the top-right corner → "Change Password".
✓ To add team members: click "Admin Panel" in the top bar → "Users" tab → fill in their name, email, role and a temporary password → click "Add User". Share the URL and their credentials with them. They can log in immediately.

🎉 You're live!

Your task manager is now a real web app, accessible from any browser, anywhere. All data is stored securely in your Supabase database and shared across your whole team in real time.

Common Questions

I got an error when running the SQL script. What do I do?▾
If you already set up Supabase for the HTML version of the app, this is normal — the tables already exist. Just skip Step 3 and continue from Step 4. Your existing data is completely safe.
The GitHub Actions step is failing (red X). What went wrong?▾
Click on the red X to see the error details. The most common causes are: (1) you uploaded the ivp-tasks folder itself instead of its contents — in that case, the file package.json won't be at the root level. Delete the repository and try Step 9 again, being careful to select files inside the folder. (2) The vite.config.js repo name doesn't match your actual GitHub repository name — double-check that spelling matches exactly.
The app loads but shows a blank page or missing styles.▾
This almost always means the repository name in vite.config.js doesn't match your actual GitHub repo name. Open vite.config.js in Notepad, fix the GITHUB_REPO_NAME value, upload the corrected file to GitHub (drag it onto the repository page and commit), and wait 2 minutes for GitHub Actions to rebuild.
The app loads but I can't log in — it says "Login failed".▾
This means your Supabase keys in supabase.js are incorrect. Go back to Step 4, copy the values again very carefully (no extra spaces), edit the file in Notepad, save it, upload it to GitHub, and wait for the rebuild. Make sure the quotes are still intact around your values.
Can I update the app after it's live?▾
Yes. If Claude gives you updated files, just upload them to your GitHub repository the same way you did in Step 9 (drag and drop onto the repository page → commit). GitHub Actions will automatically rebuild and update your live app within 2 minutes. Your data in Supabase is never affected by app updates.
Is my data safe? Can other people see our tasks?▾
Your task data lives in Supabase (a secure cloud database) — it cannot be accessed without logging in. The GitHub Pages URL is technically public but protected by the login screen. No one can see tasks without an account you create. The Supabase anon key visible in the code is designed to be public — it cannot bypass login or read data directly.
How do my team members access the app?▾
Send them your GitHub Pages URL and their login credentials (which you set when adding them in Admin Panel → Users). They visit the URL in any browser — Chrome, Safari, Edge, Firefox — no installation needed. It works on phones too.
What do I do if I forget my password?▾
As admin, go to Supabase → SQL Editor → run: SELECT public.app_set_password('YOUR-USER-ID', 'new-password123'); — your user ID is shown in the app_users table. Alternatively, any admin can reset any user's password from Admin Panel → Users → Reset PW button.
Can I use a custom domain like tasks.ivp.in?▾
Yes. First, in vite.config.js change the base line to base: '/' (remove the repo name logic). Upload that change to GitHub. Then in GitHub → Settings → Pages → Custom domain, type your domain name. Your IT/DNS team needs to add a CNAME record: tasks.ivp.in → your-username.github.io. GitHub handles the HTTPS certificate automatically within a few minutes.
